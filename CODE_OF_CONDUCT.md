# Code of Conduct

## Our Pledge

The NeMo Curator project is committed to providing a welcoming, respectful, and harassment-free experience for everyone who participates in our community — in issues, pull requests, discussions, code reviews, events, and any other project space.

## Our Standard

This project adopts the **[Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)** as its Code of Conduct. The full text is published at the link above and is incorporated here by reference.

In short:

- **Be respectful.** Treat other contributors with empathy, kindness, and patience. Assume good intent and prefer constructive feedback.
- **Be inclusive.** Welcome newcomers and people from all backgrounds. Use language that everyone can engage with.
- **Be collaborative.** Disagreements happen — keep them about the work, not the person. Defer to maintainers on direction.
- **Be accountable.** Take responsibility for mistakes, apologize when appropriate, and learn from the experience.

## Scope

This Code of Conduct applies in all official project spaces — the GitHub repository (issues, pull requests, discussions, code review), associated chat and event spaces, and any time an individual is representing the project in public. It also applies when project participants interact in private channels if that conduct affects the safety or well-being of community members.

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please report it to the maintainers so we can address it. We accept reports through either of the following channels:

- **Open a private report on GitHub** by using [Report abuse](https://github.com/contact/report-abuse) and selecting *Report content to GitHub* → *I want to report abusive content or behavior* → the relevant repository or user. GitHub forwards the report to the repository's maintainers and to GitHub Trust & Safety.
- **Contact a repository maintainer directly** via GitHub (DM or `@`-mention on a private discussion). Maintainers are listed in the repository's commit history and on the [contributors page](https://github.com/NVIDIA-NeMo/Curator/graphs/contributors).

A dedicated reporting email will be added here once confirmed with the project's open-source program office; until then, the GitHub channels above are the supported route.

All reports will be reviewed and investigated promptly and fairly. Maintainers are obligated to respect the privacy and security of the reporter of any incident.

## Enforcement

Maintainers are responsible for clarifying and enforcing this Code of Conduct and will take appropriate, fair corrective action in response to any behavior they deem inappropriate, threatening, offensive, or harmful. Enforcement follows the **[Contributor Covenant Enforcement Guidelines](https://www.contributor-covenant.org/version/2/1/code_of_conduct/#enforcement-guidelines)**, which describe a graduated set of responses ranging from a private warning to a permanent ban.
